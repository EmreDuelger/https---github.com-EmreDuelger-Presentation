Was sind KI-Agenten?

Definition

  • Software-Systeme, die ihre Umgebung wahrnehmen und autonom handeln
  • Sie verfolgen Ziele und treffen eigenständig Entscheidungen
  • Können mit Tools, APIs und anderen Agenten interagieren
  • Gehen über einfache Eingabe-Ausgabe-Modelle hinaus

Agent vs. Chatbot

Eigenschaft Chatbot Agent
Interaktion Frage → Antwort Ziel → Plan → Ausführung
Autonomie Reagiert nur auf Input Handelt eigenständig
Tools Keine oder begrenzt Nutzt externe Tools aktiv
Planung Keine Erstellt und verfolgt Pläne
Gedächtnis Kurzfristig (Kontext) Lang- und kurzfristig

Dieser Kreislauf wiederholt sich, bis das Ziel erreicht ist.

Kernkomponenten

  • LLM (Large Language Model) - Das "Gehirn" des Agenten
  • Tools / Funktionen - Ermöglichen Interaktion mit der Außenwelt
  • Gedächtnis - Speichert Kontext und Erfahrungen
  • Planung - Zerlegt komplexe Aufgaben in Schritte

Beispiel: Tool-Nutzung


// Agent erhält Aufgabe: "Wie ist das Wetter in Berlin?"

// Schritt 1: Agent erkennt - ich brauche ein Tool
agent.think("Ich muss die Wetter-API aufrufen")

// Schritt 2: Agent ruft Tool auf
const wetter = await tools.wetterAPI("Berlin")
// => { temp: 12, zustand: "bewölkt" }

// Schritt 3: Agent formuliert Antwort
agent.respond("In Berlin sind es 12°C und bewölkt.")

Typen von KI-Agenten

  • Einfache Reflex-Agenten - Reagieren direkt auf Eingaben nach festen Regeln
  • Modellbasierte Agenten - Haben ein internes Modell der Welt
  • Zielbasierte Agenten - Verfolgen definierte Ziele
  • Lernende Agenten - Verbessern sich durch Erfahrung

Multi-Agenten-Systeme

  • Mehrere Agenten arbeiten zusammen an einer Aufgabe
  • Jeder Agent hat eine spezialisierte Rolle
  • Kommunikation und Koordination zwischen Agenten

Agenten in der Praxis

Bekannte Beispiele

  • Claude Code - Programmiert autonom, nutzt Terminal und Dateisystem
  • Cursor / GitHub Copilot - KI-gestützte Code-Editoren
  • Devin - Autonomer Software-Entwickler
  • AutoGPT - Allgemeiner autonomer Agent

Anwendungsbereiche

  • Software-Entwicklung - Code schreiben, testen, deployen
  • Datenanalyse - Daten sammeln, auswerten, visualisieren
  • Kundenservice - Anfragen bearbeiten, Probleme lösen
  • Forschung - Literatur durchsuchen, zusammenfassen
  • Workflow-Automatisierung - Komplexe Geschäftsprozesse

Architektur-Pattern


// ReAct Pattern (Reason + Act)
while (!zielErreicht) {
  // 1. Beobachten
  const beobachtung = await agent.observe(umgebung);

  // 2. Nachdenken (Reason)
  const gedanke = await llm.reason(beobachtung, ziel);

  // 3. Handeln (Act)
  const aktion = await agent.act(gedanke);

  // 4. Ergebnis auswerten
  zielErreicht = await agent.evaluate(aktion);
}

ReAct kombiniert Reasoning mit Action in einem iterativen Loop.

Herausforderungen

  • Halluzinationen - Agenten können falsche Informationen generieren
  • Sicherheit - Autonome Systeme brauchen Grenzen
  • Zuverlässigkeit - Fehler können sich kaskadieren
  • Kosten - Viele LLM-Aufrufe werden teuer
  • Kontrolle - Balance zwischen Autonomie und Aufsicht

Ausblick

  • Agenten werden zuverlässiger und kostengünstiger
  • Spezialisierte Agenten für verschiedene Domänen
  • Bessere Zusammenarbeit zwischen Mensch und Agent
  • Standardisierte Protokolle für Agent-zu-Agent-Kommunikation
  • Integration in bestehende Workflows und Tools

Zusammenfassung

  • KI-Agenten sind autonome Systeme, die Ziele verfolgen
  • Sie kombinieren LLMs mit Tools und Planung
  • Der Agent-Loop (Wahrnehmen, Denken, Handeln) ist das Kernprinzip
  • Praxiseinsatz wächst rasant: Coding, Analyse, Automatisierung
  • Herausforderungen bleiben: Sicherheit, Kosten, Kontrolle

Die Zukunft gehört den Agenten.
